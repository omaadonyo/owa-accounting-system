Overview

Akatabo is a full-featured business management application built on Laravel, Livewire, and Flux UI. It provides a complete workflow for managing customers, inventory (fabrics, products, services, office rentals), creating quotations and invoices with live previews, recording payments, and generating reports — all wrapped in a clean, responsive interface with light/dark themes.

Features

Business Onboarding

4-step wizard: Business info → Address → Logo → Default document notes. Creates the Business record and auto-assigns the user as owner.

Customer Management

CRUD with name, email, address. Search, sort, paginated listing. PDF export of customer directory.

Inventory

Three tabs under one page:

  • Fabrics — Roll code, color, supplier, claimed/verified/used meters, pricing, image. Computed remaining_meters.
  • Products & Services — Name, SKU, description, buying/selling price, unit, image.
  • Office Rentals — Name, location, monthly rent, description, image.

Quotations

Create/edit with live PDF-style preview. Select customer, dates, line items from inventory or custom entry. Discount (% or fixed) and tax. Auto-numbering (QOT-XXXX). Status workflow: draft → sent → accepted / converted / rejected. Convert to invoice. PDF download with QR code.

Invoices

Same structure as quotations plus: payment recording, receipt numbers (RCT-XXXX), balance tracking (paid_amount vs total). Statuses: draft, sent, paid, overdue, cancelled, partial. PDF download with receipts summary.

Payments / Receipts

List all payments with search. Methods: cash, bank, mobile_money, card, other. CSV export. Soft-deletes.

Reports

Period filter (30d / 90d / 12m / all). Revenue, invoiced, outstanding, customer count. Payment method breakdown. Top invoices. Recent payments. PDF export.

User Management

Admin creates/manages users within the business. Roles: admin (full access) and employee (view-only for most modules). Self-deletion prevented.

Settings

  • Profile — Name, email, resend verification
  • Business — Name, email, address, logo, default invoice/quotes/receipt notes
  • Security — Password change, 2FA enable/disable/recovery codes, passkey management
  • Appearance — Light / Dark / System theme

Security Features

  • Rate limiting: login (5/min per email+IP), 2FA (5/min), passkeys (10/min)
  • Production password rules: min 12 chars, mixed case, numbers, symbols, uncompromised
  • 2FA confirmation flow (TOTP)
  • Passkey (WebAuthn) passwordless login
  • Password confirmation for sensitive actions
  • Email verification

PDF Generation

PDF templates are located in resources/views/pdf/:

  • invoice.blade.php — Full invoice with logo, items, totals, receipts summary, QR code
  • quotation.blade.php — Full quotation with logo, items, totals, QR code
  • reports.blade.php — Period-based report with stats and charts
  • customers.blade.php — Customer directory listing

All PDFs use DejaVu Sans font, inline CSS (DomPDF does not support Tailwind), and the same layout as the on-screen preview panels.

Setup

# Install dependencies
composer install
npm install

# Environment
cp .env.example .env
php artisan key:generate
# Edit .env with your database credentials

# Migrate
php artisan migrate

# Build & serve
npm run build
php artisan serve

Queue: php artisan queue:work (database driver)
Theme: Toggle light/dark/system in Appearance settings
