Overview

Akatabo is a full-featured business management application built on Laravel, Livewire, and Flux UI. It provides a complete workflow for managing customers, inventory (fabrics, products, services, office rentals), creating quotations and invoices with live previews, recording payments, and generating reports — all wrapped in a clean, responsive interface with light/dark themes.

Features

Business Onboarding

4-step wizard: Business info → Address → Logo → Default document notes. Creates the Business record and auto-assigns the user as owner.

Customer Management

CRUD with name, email, address. Search, sort, paginated listing. PDF export of customer directory.

Inventory

Three tabs under one page:

  • Fabrics — Roll code, color, supplier, claimed/verified/used meters, pricing, image. Computed remaining_meters.
  • Products & Services — Name, SKU, description, buying/selling price, unit, image.
  • Office Rentals — Name, location, monthly rent, description, image.

Quotations

Create/edit with live PDF-style preview. Select customer, dates, line items from inventory or custom entry. Discount (% or fixed) and tax. Auto-numbering (QOT-XXXX). Status workflow: draft → sent → accepted / converted / rejected. Convert to invoice. PDF download with QR code.

Invoices

Same structure as quotations plus: payment recording, receipt numbers (RCT-XXXX), balance tracking (paid_amount vs total). Statuses: draft, sent, paid, overdue, cancelled, partial. PDF download with receipts summary.

Payments / Receipts

List all payments with search. Methods: cash, bank, mobile_money, card, other. CSV export. Soft-deletes.

Reports

Period filter (30d / 90d / 12m / all). Revenue, invoiced, outstanding, customer count. Payment method breakdown. Top invoices. Recent payments. PDF export.

Public Marketplace

Dribbble-inspired dark landing page at /site showcasing all fabrics and products grouped by business. Gradient hero, glassmorphism product cards with hover animations, Browse Fabrics CTA. Pricing page at /site/pricing with plan comparison, billing toggle, FAQ accordion, and auth-aware action buttons.

Customer Quotation Requests

Public form at /site/{type}/{id}/quote where customers can request a quotation for a fabric or product. Alpine.js auto-calculates price based on product quantity or fabric length. Submissions notify the business admin via email. Admin reviews requests in /customer-quotations with options to mark as responded, delete, or Convert to Quotation which auto-creates a Quotation + QuotationItem and redirects to the edit page.

Subscription & Billing

Tiered pricing (Free / Business / Enterprise) with monthly and annual billing. Usage-based enforcement: quotas, invoices, and receipts are capped per plan with on-screen progress bars. In-app payment methods (Mobile Money, Bank Transfer, Cash) with upgrade/downgrade flow. Billing history table showing all subscriptions. Free subscription auto-assigned on business onboarding.

User Management

Admin creates/manages users within the business. Roles: admin (full access) and employee (view-only for most modules). Self-deletion prevented.

Settings

  • Business — Name, email, address, logo, default invoice/quotes/receipt notes
  • Security — Password change, 2FA enable/disable/recovery codes, passkey management

Usage Enforcement

The ChecksSubscriptionLimits trait counts records since the subscription start date. Quotation/invoice creation pages block new records when the limit is reached (toast with upgrade prompt). Dashboard displays usage progress bars.

Payment Flow

  • Mobile Money — MTN or Airtel; user sends payment and enters transaction ID
  • Bank Transfer — User transfers to provided bank details and enters reference
  • Cash — In-person payment with optional reference
  • Free plan upgrades are instant (status=active); paid subscriptions start as pending until confirmed

Upgrade Modal

The billing page uses custom searchable select dropdowns (see below) for Payment Method and Mobile Money Provider fields inside the upgrade modal. Downgrading to Free skips payment validation. After subscription changes, the page redirects to force a clean component re-initialization.

Public Marketplace

The /site route renders a dark-themed marketplace landing page with a gradient hero, glassmorphism cards, and hover animations. Products and fabrics are grouped by business. The page is fully public (no authentication required).

Customer Quotation Flow

  1. User browses items on /site and clicks “Request Quote”
  2. A form pre-loaded with item details auto-calculates the price (quantity for products, length for fabrics) via Alpine.js
  3. On submit, a CustomerQuotation record is created and the business admin receives an email notification
  4. Admin reviews requests in /customer-quotations, can mark as responded, delete, or convert to a full Quotation

Security Features

  • Rate limiting: login (5/min per email+IP), 2FA (5/min), passkeys (10/min)
  • Production password rules: min 12 chars, mixed case, numbers, symbols, uncompromised
  • 2FA confirmation flow (TOTP)
  • Passkey (WebAuthn) passwordless login
  • Password confirmation for sensitive actions
  • Email verification

PDF Generation

PDF templates are located in resources/views/pdf/:

  • invoice.blade.php — Full invoice with logo, items, totals, receipts summary, QR code
  • quotation.blade.php — Full quotation with logo, items, totals, QR code
  • reports.blade.php — Period-based report with stats and charts
  • customers.blade.php — Customer directory listing

All PDFs use DejaVu Sans font, inline CSS (DomPDF does not support Tailwind), and the same layout as the on-screen preview panels.

Custom Searchable Select

Replaces native <select> elements (previously flux:select) that were glitchy inside Flux modals. Built with vanilla JavaScript (no framework dependency).

Features

  • Type-to-search filtering of options
  • Keyboard navigation: Escape closes, Enter auto-selects the only visible match
  • Outside-click and Tab closes the dropdown
  • Syncs with Livewire wire:model via a hidden <select> element, dispatching both input and change events
  • MutationObserver auto-initializes new instances when Livewire morphs the DOM
  • Checkmark indicator on the selected option
  • Indigo focus ring on the trigger when open
  • Full dark mode support

Usage

<div class="custom-select relative">
    <button type="button" data-cs-trigger class="...">
        <span data-cs-display>Selected option</span>
        <svg>...chevron-down...</svg>
    </button>
    <div data-cs-dropdown class="hidden absolute ...">
        <div>
            <input type="text" data-cs-search placeholder="Search...">
        </div>
        <div data-cs-options>
            <button type="button"
                data-cs-option
                data-cs-value="value"
                data-cs-label="Label">Label</button>
        </div>
    </div>
    <select wire:model="property" class="sr-only">
        <option value="value">Label</option>
    </select>
</div>

Files

  • resources/js/custom-select.js — ES module with the CustomSelect class
  • resources/css/app.css — .cs-selected checkmark pseudo-element and .cs-open trigger ring
  • resources/js/app.js — imports the module

Setup

# Install dependencies
composer install
npm install

# Environment
cp .env.example .env
php artisan key:generate
# Edit .env with your database credentials

# Migrate & seed
php artisan migrate
php artisan db:seed --class=PlanSeeder
php artisan subscriptions:assign-free

# Build & serve
npm run build
php artisan serve

Queue: php artisan queue:work (database driver)
Theme: Toggle light/dark/system in Appearance settings
